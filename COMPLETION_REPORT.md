## 🔐 SECURITY

### Environment Variables (Public)
- `NEXT_PUBLIC_*` variables are intentionally public
- This is normal for Firebase frontend apps
- Your actual DB security = Firestore Rules

### Firestore Rules
```javascript
// Development (in .env.local already)
match /leads/{document=**} {
  allow read, write: if true;
}

// Production (You'll change this)
match /leads/{document=**} {
  allow create: if true;
  allow read, write: if false;
}
```
